### 为什么要与我们连接？

加入社区可以带来许多收获，帮助你真正产生影响：

- **协作共创：** 我们欢迎不同背景与经验的贡献者，你的独特视角就是宝贵财富。
- **认可肯定：** 所有努力都不会被忽视。我们乐于庆祝每一份贡献，你还可以在我们的[贡献者图表](##Contributors)中看到所有伙伴。
- **透明开放：** 我们坚持公开决策，所有策略选择都遵循 [行为准则](https://github.com/MemMachine/MemMachine/blob/main/CODE_OF_CONDUCT.md)。
- **共同成长：** 无论大小的贡献，都在共同打造真正出色的成果。

## 参与方式

- **代码贡献：** 在 [GitHub 仓库](https://github.com/MemMachine/MemMachine) 查找问题并提交修复。
- **文档改进：** 优化指南、纠正错别字或翻译内容。
- **设计 / 用户体验：** 帮助塑造项目的外观与交互。
- **测试 / 质量保障：** 反馈缺陷、测试新功能。
- **推广宣传：** 通过博客或社交媒体传播 MemMachine。

## 行为准则

### 我们的承诺

作为社区成员、贡献者与维护者，我们承诺为每位参与者提供一个无骚扰、友善且包容的环境，不因年龄、体型、能力、种族、性别认同与表达、经验水平、国籍、外貌、宗教信仰、性取向等差异而区别对待。

### 行为准则

以下行为有助于营造积极的社区氛围：

- 对他人保持共情与善意。
- 尊重不同的意见、观点和经历。
- 主动给予并坦然接纳建设性的反馈。
- 勇于承担责任、向受影响的人致歉，并从错误中学习。
- 关注整体社区的利益，而非仅仅考虑个人。

### 不当行为

以下行为不可接受：

- 使用带有性暗示的语言或图像，或进行任何形式的骚扰。
- 恶意挑衅、侮辱或发表贬损性言论，以及人身或政治攻击。
- 公开或私下的骚扰行为。
- 未经允许公开他人的私人信息，如地址或邮箱。
- 其他被普遍认为不适合专业环境的行为。

### 举报指引

若遇到虐待、骚扰或其他不可接受的行为，请通过 GitHub 的私有渠道联系项目团队，可在仓库的 [Security](https://github.com/MemMachine/MemMachine/security) 页面提交报告。我们会及时且公平地审查所有投诉。

对于安全问题，我们使用 GitHub 的私密漏洞报告功能。如果你发现 MemMachine 存在安全隐患，**请勿创建公开 Issue**，而应在 [Security](https://github.com/MemMachine/MemMachine/security) 页面点击 “Report a privately reported vulnerability” 进行私密报告。

项目负责人必须尊重举报者的隐私与安全。

### 执行

项目负责人将遵循 Contributor Covenant 的执法指南处理每一次违规事件。如若负责人违反行为准则，也将面临与其他社区成员相同的处理结果。

### 致谢

本行为准则改编自 Contributor Covenant 2.1 版本，原文见：[https://www.contributor-covenant.org/version/2/1/code_of_conduct.html](https://www.contributor-covenant.org/version/2/1/code_of_conduct.html)。
